You are a senior full-stack developer and system designer.

Add a COMPLAINT / GRIEVANCE BOX module to a School Management System.

========================
MODULE OVERVIEW
========================

This module allows Students and Teachers to submit complaints
and allows Admin to manage, respond, and resolve them.

========================
USER SIDE (STUDENT / TEACHER)
========================

Create a Complaint Form with fields:
- Complaint Title
- Complaint Category
  (Academic, Teacher, Student, Infrastructure, Exam, Fee, Other)
- Description (textarea)
- Priority (Low, Medium, High)
- Anonymous submission option (checkbox)
- File attachment (optional)

Auto-capture:
- User Role (Student / Teacher)
- User ID
- Class (for students)
- Timestamp

On submit:
- Save complaint to database
- Set status = "Open"
- Show success message

========================
ADMIN SIDE (WEB PANEL)
========================

Create a Complaint Management Dashboard where Admin can:

- View all complaints in a table
- See:
  - Title
  - Submitted By (hidden if anonymous)
  - Role
  - Category
  - Priority
  - Status
  - Date

Filters:
- Role (Student / Teacher)
- Status (Open, In Progress, Resolved, Closed)
- Priority
- Category
- Date range

Actions:
- View full complaint details
- Reply to complaint
- Change status
- Add internal admin notes
- Assign complaint (optional)

========================
COMPLAINT DETAILS VIEW
========================

Show:
- Complaint information
- Timeline (Submitted → Replied → Resolved)
- Admin responses
- Status updates

========================
ANALYTICS (OPTIONAL)
========================

Show:
- Total complaints
- Open vs Resolved
- High priority complaints
- Category-wise distribution
- Average resolution time

========================
TECHNICAL REQUIREMENTS
========================

Backend:
- Node.js + Express
- MongoDB schema for complaints
- Secure role-based access
- File upload handling
- Status lifecycle management

Frontend:
- React + Tailwind CSS
- Clean modern UI
- Status badges & colors
- Responsive forms
- Role-based views

========================
SECURITY RULES
========================

- Users can see ONLY their own complaints
- Admin can see ALL complaints
- Anonymous complaints hide identity in UI
- Proper validation & error handling

Build the COMPLETE Complaint module with production-ready code.
Do NOT skip logic.
